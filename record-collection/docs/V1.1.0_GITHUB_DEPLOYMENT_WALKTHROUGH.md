# Lititz BMX Record Collection v1.1.0 — GitHub Deployment Walkthrough

## Before copying files

1. Download and extract `lititz-bmx-record-collection-v1.1.0-final-deployment.zip`.
2. Confirm the extracted package contains one top-level folder named `record-collection`.
3. In GitHub Desktop, select `Article134-tech/lititzbmx-docs` and branch `main`.
4. Click **Fetch origin**. Pull any remote changes before beginning.
5. Open the local repository folder and make a temporary backup of the current `record-collection` folder.

## Replace only the controlled folder

1. Copy the extracted `record-collection` folder into the root of the existing local `lititzbmx-docs` repository.
2. Choose **Replace the files in the destination** when prompted.
3. Do not delete or replace any other top-level folder or root file.
4. Return to GitHub Desktop.

## Review before committing

Confirm all of the following:

- every changed path begins with `record-collection/`;
- no file under `campaigns/`, `learning-resources/`, `brand-assets/`, `data/`, `reference/`, or `standards/` changed;
- the Record Collection banner, category covers, navigation thumbnails, README files, and v1.1.0 documentation appear in the change list;
- no private source supplement or temporary build folder appears.

Use this commit summary:

`Expand Lititz BMX Record Collection to v1.1.0`

Commit to `main`, then click **Push origin**.

## Post-deployment browser checks

Open the repository on GitHub and verify:

1. `record-collection/README.md` displays the 21-dossier banner.
2. The category counts show 8 Fireside BMX Chat, 4 Pump Track Chat, and 9 Unboxing dossiers.
3. Each category cover opens its visual dossier index.
4. Every dossier card opens the correct dossier README.
5. The four “The Boy” derivative clips remain inside that dossier and do not appear as separate dossier cards.
6. The Warwick public-comment record states that it is a rehearsal/reconstruction, not meeting footage.
7. The Damian Fulton Samaritan’s Purse record appears under Fireside BMX Chat.
8. The Bill Allen / RAD record states that no transcript was supplied and contains no Reynolds Racing transcript.
9. Images render without broken links.
10. `record-collection/docs/V1.1.0_DEPLOYMENT_VALIDATION.md` is present.

## Optional root README correction

The root repository README should be changed only when its existing Record Collection entry is missing or still says seven dossiers. Use `record-collection/docs/ROOT_README_INDEX_ENTRY.md` and make that root README update as a separate commit.

Recommended separate commit summary:

`Update Record Collection repository index`

## Rollback

If any unexpected path changed or rendering is broken, do not make unrelated repairs in the same commit. Revert the v1.1.0 commit or restore the backed-up `record-collection` folder, identify the problem, and redeploy the corrected controlled package.
